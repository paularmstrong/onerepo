## Quick start

<Steps>

1. **Install the CLI**

   First, install oneRepo's `one` command into your path:

   <Tabs syncKey="pkgmanager">

    <TabItem label="npm">

   ```sh title="Install via npm"
   npx --package=onerepo one install
   ```

    </TabItem>

    <TabItem label="yarn">

   ```sh title="Install via Yarn"
   yarn dlx --package=onerepo one install
   ```

    </TabItem>

    <TabItem label="pnpm">

   ```sh title="Install via pnpm"
   pnpm --package=onerepo dlx one install
   ```

    </TabItem>

   </Tabs>

2. **Create your oneRepo**

   Next, use the `one create` command to initialize into an existing or a new repository.

   ```sh title="Initialize oneRepo in a new or existing repository"
   one create
   ```

3. **Bring your own Package Manager**

   oneRepo is compatible with all major package managers for Node.js.

   <Tabs syncKey="pkgmanager">

    <TabItem label="npm">

   ```ansi title="Choose your own package manager"
   [32m?[0m Which package manager would you like to use?
   [34m‚ùØ npm[0m
    pnpm
    yarn
   ```

    </TabItem>

    <TabItem label="yarn">

   ```ansi title="Choose your own package manager"
   [32m?[0m Which package manager would you like to use?
    npm
    pnpm
    [34m‚ùØ yarn[0m
   ```

    </TabItem>

    <TabItem label="pnpm">

   ```ansi title="Choose your own package manager"
   [32m?[0m Which package manager would you like to use?
    npm
   [34m‚ùØ pnpm[0m
   yarn
   ```

    </TabItem>

   </Tabs>

4. **Configure your repo**

   Modify the root configuration file, `./onerepo.config.ts`, for your repo‚Äôs specific needs.

   Refer to the full [configuration documentation](/docs/config/) for more information.

   <Tabs syncKey="jsflavor">

    <TabItem label="ES Module">

   ```ts title="./onerepo.config.js" {3-5}
   export default {
   	root: true,
   	codeowners: {},
   	tasks: {},
   	// And much more!
   };
   ```

   </TabItem>

   <TabItem label="TypeScript">

   ```ts title="./onerepo.config.ts" {5-7}
   import type { Config } from 'onerepo';

   export default {
   	root: true,
   	codeowners: {},
   	tasks: {},
   	// And much more!
   } satisfies Config;
   ```

   </TabItem>

   <TabItem label="Common JS">

   ```ts title="./onerepo.config.cjs" {3-5}
   module.exports = {
   	root: true,
   	codeowners: {},
   	tasks: {},
   	// And much more!
   };
   ```

    </TabItem>

   </Tabs>

5. **Ready to go!**

   That‚Äôs it! Change into your monorepo‚Äôs root directory and use the `one` CLI.

   ```sh
   cd my-monorepo
   one --help
   ```

   ```
   one <command> [options]
   one workspace <workspace-name> <command> [options]

   Commands:
     one tasks     Run tasks against repo-defined lifecycles. This
                   command will limit the tasks across the affected
                   workspace set based on the current state of the
                   repository.
     one hooks     Manage git repository hooks
     one graph     Run core Graph commands
     one generate  Generate files, folders, and workspaces from
                   templates.    [aliases: gen]
   # ‚Ä¶
   ```

</Steps>

## Manually integrating oneRepo

<Steps>

1.  **Install dependencies**

    Install dependencies using your package manager of choice.

    <Tabs syncKey="pkgmanager">

     <TabItem label="npm">

    ```sh title="Install via npm"
    npm install --save-dev onerepo
    ```

     </TabItem>

     <TabItem label="yarn">

    ```sh title="Install via Yarn"
    yarn add --dev onerepo
    ```

     </TabItem>

     <TabItem label="pnpm">

    ```sh title="Install via pnpm"
    pnpm add --save-dev onerepo
    ```

     </TabItem>

    </Tabs>

1.  **Install the CLI**

    Install the CLI into your system. This will create a command, `one`, that can be run from anywhere in your path. It will always pick up a local version of `onerepo`, if it exists, otherwise fall back on the current version.

    <Tabs syncKey="pkgmanager">

     <TabItem label="npm">

    ```sh title="Install via npm"
    npm exec one install
    ```

     </TabItem>

     <TabItem label="yarn">

    ```sh title="Install via Yarn"
    yarn one install
    ```

     </TabItem>

     <TabItem label="pnpm">

    ```sh title="Install via pnpm"
    pnpm one install
    ```

     </TabItem>

    </Tabs>

1.  **Configure workspaces**

    Configure your package manager to work with workspaces

    <Tabs syncKey="pkgmanager">

      <TabItem label="npm">

        ```json title="package.json" {4}
        {
         "name": "my-monorepo",
         "private": true,
         "workspaces": ["apps/*", "modules/*"]
        }
        ```

      </TabItem>

      <TabItem label="yarn">

        ```json title="package.json" {4}
        {
          "name": "my-monorepo",
          "private": true,
          "workspaces": ["apps/*", "modules/*"]
        }
        ```

     </TabItem>

     <TabItem label="pnpm">

    ```yaml title="pnpm-workspace.yaml"
    packages:
      - 'apps/*'
      - 'modules/*'
    ```

     </TabItem>

    </Tabs>

1.  **Configure**

    Create a configuration file at the root of your repository. This may be either JavaScript or TypeScript.

    Refer to the full [configuration documentation](/docs/config/) for more information.

    <Tabs syncKey="jsflavor">

      <TabItem label="ES Module">

        ```ts title="./onerepo.config.js"
        export default {
          root: true,
        };
        ```

      </TabItem>

      <TabItem label="TypeScript">

        ```ts title="./onerepo.config.ts"
        import type { Config } from 'onerepo';

        export default {
          root: true,
        } satisfies Config;
        ```

      </TabItem>

      <TabItem label="Common JS">

        ```ts title="./onerepo.config.cjs"
        module.exports = {
          root: true,
        };
        ```

     </TabItem>

    </Tabs>

</Steps>
